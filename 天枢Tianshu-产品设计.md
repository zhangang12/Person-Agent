# 天枢 Tianshu — 全能研发智能体产品设计（v0.2）

> 修订说明：v0.1 把研发拆成固定的九段流水线——那是「工作流」，不是「智能体」。
> v0.2 推翻它：**天枢是一个你直接对话、自己判断该调什么能力的全能智能体。一个对话框，无限能力，由意图驱动，而非被排成顺序。** 交互采用苹果气泡式对话风格。
> 技术底座不变：**OpenCode `serve`** 作引擎（本就是通用 agent），天枢是它的对话式外壳。

---

## 1. 定位（一句话）

> 一个跑在你桌面、用你内网模型的**全能 AI 同事**。你说人话，它自己决定读代码 / 改代码 / 查需求 / 评审 / 查库 / 写文档……需要时以「建议气泡」轻轻提示下一步，**绝不强推流程**。

---

## 2. 为什么不是固定工作流（直接回应核心判断）

| | 固定工作流（v0.1，已弃） | 全能智能体（v0.2） |
|---|---|---|
| 主语 | 系统驱动人，按阶段推进 | 人驱动系统，按意图调度 |
| 路径 | 需求→设计→…→运维，线性 | 任意顺序、任意跳转、随时回头 |
| 能力 | 是「阶段」，被顺序排列 | 是「技能」，被意图随时调用 |
| 体验 | 填表式、被框住 | 对话式、随机应变 |
| 适配现实 | 研发本不线性，流程僵硬 | 你可能先评审、再改需求、又查段历史 SQL |

要点：**LLM 本就是通用的**，把它锁进阶段机反而阉割了随机应变。能力依然全在，只是从「阶段」变回「技能」。

---

## 3. 核心理念（四条）

1. **一个对话，无限能力**：单一智能体 + 单一连续对话，不是 N 个面板、N 个工具。
2. **意图驱动，不是流程驱动**：你表达意图，它编排手段；没有必经之路。
3. **你的模型、你的数据、你的机器**：OpenCode 接内网私有化模型，代码/SQL/客户信息不出网。
4. **能力可扩展**：加一种技能 = 加一个插件，对话核心不动。

---

## 4. 能力（随取随用的技能，不是阶段）

下面是「它能做什么」，**任意组合、任意顺序、对话中一句话触发**，彼此无固定先后。

| 分组 | 技能 | 怎么触发 |
|---|---|---|
| 理解 | 代码问答、需求落点定位、SQL/Mapper 解读、调用链/影响面 | 「这段干嘛的」「改这个牵连哪些表」 |
| 修改 | 代码生成、重构、Bug 修复 | 「把这块抽个公共方法」「这个空指针修一下」 |
| 质量 | 代码评审、单测生成、安全检查（SQL 注入/事务/金额单位） | 「看下这个改动有没有坑」 |
| 交付 | 构建解读、发布单 + 回滚预案、变更影响 | 「准备投产，给个发布方案」（按需，不强制） |
| 运维 | 日志/告警分析、根因定位、性能诊断 | 「这条告警什么原因」 |
| 知识 | 文档/注释生成、维护 wiki/glossary（连续记忆） | 「把这个模块补个文档」 |
| 横切 | Git 辅助、查内网系统（GitLab/Jira/只读库，经 MCP） | 「拉一下这个工单」 |

> 与你已有信贷 wiki 咬合：定位/评审直接复用 `glossary`、`capability-map`、`review`，前期投入立刻变现。但它们是「随手能调的技能」，不是「必须走的阶段」。

## 5. 桌面形态：全局输入 + 并行玻璃卡

- **全局输入框**：像 Spotlight / Raycast，`⌘Space` / 全局热键随处唤起，悬浮在桌面之上；选中代码也能直接喂进去。
- **一句话开一张卡**：每输入一个任务，生成一张半透明玻璃卡，独立执行——不打断你手头的事。
- **多卡并行**：可同时开 N 张卡跑不同任务（评审、补测、定位、发布单…），互不阻塞。**对应 OpenCode 多会话并发——每张卡 = 一个 `session`。**
- **独立可拖动窗口**：每张卡是一个无边框透明窗口，可拖到屏幕任意位置、自由摆放 / 堆叠 / 收起；背后透出的是你真实的桌面与应用，**产品本身不带壁纸 / 背景**。
- **卡片即会话**：卡上显示任务标题、状态（运行 / 完成 / 排队）、会话号、实时活动行、用时。
- **实时流式回写**：`/event` 把每张卡的进度分别推回，卡片各自更新，谁先完成谁先亮。
- **建议气泡**：卡内答完轻轻给「可选下一步」（点或不点都行，不是流程）。
- **斜杠命令**：`/review` `/locate` `/test` 给老手快捷入口，永远可选。
- **人在回路**：写 / 执行类操作前轻量确认；只读默认放行。

## 6. 技术底座（与 v0.1 的关键区别）

```
   你（自然语言 / 选区 / 斜杠命令）
        ↓  苹果气泡对话界面
   天枢外壳：意图→技能 轻路由 · 技能注册表 · 权限策略 · 连续记忆
        ↓  opencode serve HTTP API + /event
   OpenCode（通用 agent：会话/工具/事件/权限）
        ↓
   内网私有化大模型（数据不出网）
```

- 引擎：OpenCode serve，本就是通用 agent，不重造。
- 天枢做的：**全局输入 + 玻璃卡前端** +「技能注册表」（技能 = 可被意图调用的能力插件）+ 权限策略 + 连续记忆。
- **并行的实现**：host 同时 `POST /session` 开多个会话，各自 `POST /session/:id/message`，单条 `/event` 流按 `sessionID` 多路分发到对应卡片。一张卡崩了不影响其它卡。
- **多窗口架构**：不是一个大窗口里画卡片，而是**每张卡 = 一个独立的透明无边框 OS 窗口**（Tauri 多窗口），各自可拖动 / 置顶 / 关闭；窗口级透明 + 系统毛玻璃由 OS 提供，背后透出真实桌面。全局输入框是另一个常驻小窗口。
- **和 v0.1 最大不同**：编排器不是「阶段状态机」，而是「意图 → 技能」的轻路由——多数时候直接把你的话交给通用 agent，技能只是给它更强的手、给你更顺的入口。

## 7. 设计语言（透明玻璃拟态）

- **玻璃卡（材质已定）**：**浅色磨砂玻璃 + 深色字**（macOS 浅色 vibrancy）——白色约 60% 不透明度 + 毛玻璃，壁纸透成淡彩、深色字清晰；柔和大圆角、白色高光描边。避免纯白不透明（那是 bug，不是设计）。
- **悬浮层次**：卡片像 Stage Manager 错落悬浮，轻微景深，不铺满、不抢焦。
- **环境光**：壁纸透出的柔色光晕作背景，克制冷色为主，暗色自动适配。
- **动效**：卡片吹气泡式入场、运行态呼吸 / 转圈、完成态轻定格、谁先完成谁先亮。
- **无产品背景**：卡片透出的是你真实桌面 / IDE，**不是内置壁纸**；材质要在任意背景上都清楚（靠系统 vibrancy）。
- **真·透明靠桌面端实现**：Tauri / Electron 的窗口透明 + 系统毛玻璃（macOS vibrancy、Windows Acrylic / Mica）。claude 里的概念图用 `backdrop-filter: blur` 近似，编辑器背景仅为"示意你的真实桌面"。
- **一句话原则**：科技感来自「通透 + 并行的秩序感」，不是霓虹与 HUD。

## 8. MVP 与风险

**MVP**：全局输入框 + 能并行开 2–3 张玻璃卡 + 三个最常用技能（定位 / 评审 / 补测）+ 建议气泡 + 常驻热键。
底座直接用已有的 `desktop-agent` 脚手架——capabilities 改造成「意图可调的技能」、host 支持**并发多会话**、`/event` 按 `sessionID` 分发到卡片，**去掉任何强制顺序**，再套上玻璃卡前端。

| 风险 | 对策 |
|---|---|
| 模型质量参差 | 结论带出处 + 人在回路；关键问题可切更强模型 |
| 误操作 | 只读默认放行、写/执行需确认、危险命令黑名单、全程审计 |
| 「全能」过度承诺 | 先把研发对话做透（最强场景），再谈泛办公 |
| 能力变多变乱 | 技能插件化，意图路由按需加载，不堆面板 |

> **天枢 = OpenCode（通用引擎）+ 苹果气泡对话（交互）+ 随取随用的技能（能力）+ 连续记忆（你的 wiki）。**
