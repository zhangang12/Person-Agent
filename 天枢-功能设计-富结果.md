# 天枢 · 功能设计：富结果（让输出能用）

> 一句话：AI 的回复里只要有**可操作的产物**（代码 / diff / 文件引用 / 命令），就给它一个**一键动作**，把输出直接闭环回你的代码与工具。
> 铁律：所有"动手"动作仍走 **opencode（编辑/运行经权限）** 或**本地（打开编辑器）**；天枢只做识别、渲染、调度，不自己改文件。

---

## 1. 设计原则
1. **看见即可用**：能操作的产物旁边永远有动作按钮，不用复制粘贴到别处。
2. **一键闭环**：diff → 应用到工作区；命令 → 运行看输出；`文件:行` → 编辑器定位。
3. **人在回路**：写/执行类动作经内联权限确认（沿用现有机制），危险不自动。
4. **失败可恢复**：应用不干净 / 命令报错 → 明确提示 + 重试 / 手动打开，不静默。
5. **贴合玻璃卡**：动作条轻量、克制，符合《设计规范》令牌，不喧宾夺主。

---

## 2. 识别哪些产物（5 类块）

| 类型 | 识别规则 | 主动作 |
|---|---|---|
| **Diff 块** | 围栏 ```` ```diff ````，或统一 diff 特征（`@@`、`+/-` 行），或 AI 标注"应用以下改动" | **应用** / 复制 / 打开 |
| **命令块** | 围栏 ```` ```bash/sh/shell/powershell ```` | **运行** / 复制 |
| **代码块**（其它语言） | 围栏 ```` ```lang ```` | 复制 / （若含目标文件名）应用 |
| **`文件:行` 引用** | 正则 `路径.扩展名:行号`（已实现 linkify） | **在编辑器打开** |
| **结构化结果**（评审/定位等） | 能力以 `json_schema` 返回（如 必改/建议、files/tables/apis） | 渲染为 findings 卡，每条可点 `文件:行` |

> 保守策略：只有**围栏 shell 块**才给"运行"；行内 `code` 只复制，避免误判误跑。

---

## 3. 每类块的设计

### 3.1 代码块
- 顶部一条**动作条**：左=语言标签（`java`），右=`复制`（点后 1.5s 变"已复制 ✓"）。
- 等宽、可横向滚动（已实现）；语法高亮（用轻量高亮，离线内置）。

### 3.2 Diff 块（★核心）
- 渲染：`+` 行绿底、`-` 行红底、`@@` 头灰色；左侧可选行号。
- 动作条：**`应用`**（主按钮，签名色）· `复制` · `在编辑器打开`。
- 多文件 diff：按文件分段，每段一个"应用本文件"，顶部一个"全部应用"。

### 3.3 命令块
- 动作条：**`运行`**（主按钮）· `复制`。
- 运行后：命令块下方展开**输出区**（折叠/展开），流式追加 stdout。

### 3.4 `文件:行` 引用
- 行内链接（签名色虚线下划线）；点击 → 本地编辑器在该行打开。
- 编辑器命令可配（`code -g "{file}:{line}"` 默认；IDEA 用 `idea64 --line {line} "{file}"`）。

### 3.5 结构化结果（findings）
- 渲染成小卡：每条带严重度徽标（必改=红 / 建议=黄 / 可忽略=灰）+ 说明 + 可点 `文件:行`。
- 底部"全部转为 TODO / 复制清单"（后续）。

---

## 4. 核心流程：Diff 一键应用

> 关键约束：天枢**不自己写文件**，而是**让会话里的 agent 用编辑工具应用**（经权限），符合外层封装铁律。

```
点【应用】
  → 天枢向当前会话发一条指令：
     "把下面这段 diff 应用到 <file>（用编辑工具实际改文件，不要只解释）：\n<diff>"
  → agent 调 edit/patch/write 工具
  → 写操作触发内联权限条 → 你点【允许一次】
  → agent 改文件、回报结果
  → 天枢拉 GET /session/:id/diff 或读回报 → 卡片把该 diff 块标记【已应用 ✓】
     并可在块下展示"实际改动"（来自 session diff）
```

**应用按钮状态机**：`待应用` → `应用中…`（可能弹权限） → `已应用 ✓` / `应用失败 ✗`。
**失败**（diff 不干净/文件不符/冲突）：标红 + 一句原因 + `让 AI 重试` / `在编辑器打开手动改`。

---

## 5. 核心流程：运行命令

```
点【运行】
  → 天枢向会话发："运行 shell 命令 `<cmd>` 并返回输出。"
  → agent 调 bash 工具 → 权限条（bash 需确认）→ 你允许
  → stdout 流式回来 → 写进命令块下方的"输出区"
```
**状态**：`运行` → `运行中…`（转圈，可【停止】）→ 输出展开 / `运行失败`（显示报错）。
**安全**：默认必须确认；危险命令（rm/format/drop…）即使点运行也先红字警示。

---

## 6. 状态与反馈（统一规范）
| 动作 | idle | loading | done | error |
|---|---|---|---|---|
| 复制 | `复制` | — | `已复制 ✓`（1.5s 回落） | `复制失败` |
| 应用 | `应用` | `应用中…` | `已应用 ✓` | `应用失败 ✗`（原因+重试） |
| 运行 | `运行` | `运行中…`（可停止） | 输出展开 | `运行失败`（报错） |
| 打开 | `文件:行` 链接 | — | 编辑器跳转 | `打不开`（提示配 editorCmd） |

---

## 7. 视觉（配合《设计规范》）
- 块用玻璃卡内的"次级面板"：`--composer-bg` 底 + `--composer-border` 边 + 圆角。
- diff 行底色用低饱和红/绿（不刺眼，暗色主题下也清楚）。
- 动作条按钮小而克制：主动作（应用/运行）用签名渐变，其余幽灵按钮。
- 反馈用微动效（已复制淡入淡出、应用中转圈、已应用对勾定格）。

---

## 8. 边界情况
- **超大 / 多文件 diff** → 折叠预览 + "全部应用"分文件执行；进度提示。
- **diff 与当前文件不一致** → agent 应用失败 → 提示"代码已变，建议让 AI 基于最新代码重生 diff"。
- **命令长跑 / 卡住** → 输出区显示运行中 + 【停止】（走 `/session/:id/abort`）。
- **相对路径** → 以该卡的项目目录（serve cwd）解析 `文件:行`。
- **离线高亮** → 语法高亮用内置轻量方案，不连公网。

---

## 9. 实现要点（遵守外层封装铁律）
- **识别/渲染**：在 `card.html` 的 Markdown 渲染里加 块类型识别 + 动作条（纯前端）。
- **应用 / 运行**：通过 `card-send` 向**同一会话**发结构化指令，让 agent 用工具做；权限走现有内联条。**天枢不直接写文件/跑命令。**
- **打开编辑器**：本地动作，`shell`/`exec` 调 `editorCmd`（已有雏形），与 opencode 无关。
- **实际改动**：用 `GET /session/:id/diff` 取本轮改了什么，回填到卡片。
