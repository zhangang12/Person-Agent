# 浏览器 + Agent 应用场景

> 内嵌浏览器(WebContentsView)× 对话 Agent × Record&Replay 技能,三者组合出的落地玩法。
> 语境:内网私有化、数据不出门、信贷/柜面类业务系统 + 前端研发。

## 一、研发调试类(已闭环)

### 1. 前端 Bug 一键取证 → Agent 修 → 自动回放验证
浏览器里复现问题 → 「录制」抓复现路径 → 「AI 分析」把 URL+控制台报错+DOM+录制时间线打包给 Agent → Agent 改代码 → 点「验证」自动回放录制,diff 修复前后报错/网络异常,出 PASS/FAIL/SUSPICIOUS 报告(带断言/覆盖率命中/影响半径/自审四类硬证据)。
**价值:修没修好不靠嘴说,靠回放证据。**

### 2. 回归检查:改完代码,把核心路径技能跑一遍
把「登录→查客户→提交申请」「导出报表」等核心路径各录成技能。每次改完前端,对 Agent 说"把技能库里带⭐的都跑一遍",Agent 逐个 `skill_run`,汇总每条 PASS/FAIL。
**价值:内网没有 CI/E2E 基建也能有回归测试,零脚本编写。**

### 3. 接口联调观察
浏览器网络面板(含 200 业务异常识别:HTTP200 但 code!=0)+ Agent 分析,直接问"刚才这次提交为什么失败",Agent 读网络时间线+控制台因果链回答。

## 二、业务操作自动化类(技能核心场景)

### 4. 高频重复录入 → 录一次,参数化复用
柜面/信贷系统里每天重复填的表单(如台账登记、额度录入):录一遍,把客户号/金额勾成参数。之后技能库点 ▶ 填两个参数就跑;或对 Agent 说"用台账登记技能,客户号 6222…,金额 5 万"。
**价值:五分钟的表单流程变成十秒钟一句话。**

### 5. 批量操作:Agent 循环调技能
"把这个 Excel 里 20 个客户挨个用「查额度」技能查一遍,汇总成表" —— Agent 读附件取数 → 循环 `skill_run` 带不同参数 → 汇总结果回话。
**价值:技能(单次流程)× Agent(编排循环)= 批处理,不用写任何 RPA 脚本。**

### 6. 定时巡检(配合触发器/定时任务)
把「打开监控页→筛选今日→截图」录成技能,定时触发让 Agent 跑并把结果/截图发邮件。
**价值:内网系统没有开放 API 也能做自动巡检 —— UI 就是 API。**

## 三、信息获取类

### 7. 内网系统查数据,Agent 消化后回答
很多内网系统(老 OA、工单、Wiki)没有接口。Agent 用技能跑到目标页,再用 `browser_get_text` 取正文消化,直接回答"今天有几个待审批"。
**价值:把"只能人肉点开看"的系统变成 Agent 可读的数据源。**

### 8. 跨系统串联
"从信贷系统查客户状态(技能A),再到影像系统看资料齐不齐(技能B),都好了就用邮件工具通知客户经理" —— 技能是原子能力,Agent 是编排器,邮件/DB/文档工具是旁路。
**价值:一句话驱动跨三个系统的工作流。**

## 四、验证与守护类

### 9. 修复验证的"影子路径"
Agent 改后端代码后,跑对应前端技能验证端到端行为(不只看单测),配合 200 业务异常检测确认接口真正常。

### 10. 升级/发版前后对照
版本升级前把关键路径技能各跑一遍留报告,升级后再跑,diff 两次结果 —— 免费的发版冒烟测试。

---

## 能力边界(诚实声明)
- 选择器继承录制时的健壮性(id/data-test 优先,多候选 fallback),但页面大改版后技能可能需要重录;
- 登录态:回放恢复录制时的 cookie/storage,长期技能建议运行前先在浏览器里登录;
- 参数只落在**输入框步**(防注入设计),藏在 URL/点击目标里的变量暂不支持;
- 同环境复用(host 变量化 dev/uat/prod 在 phase-2)。
