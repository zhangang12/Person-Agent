BocomHermes 的「研发流程功能」路线图在 `docs/研发功能路线图.md`(2026-06-24 提交 `7788455`),面向研发团队**内部流程**的功能(区别于安全/CI/可观测这类隐性机制)。5 个旗舰:①跨双仓影响面雷达 ②前后端契约对账单 ③录制即用例闭环 ④需求↔代码追溯台账 ⑤事故指纹库。

**当前正在做的是「需求分析自动化」**(2026-06-26 起)。

**关键纠偏(必须记住):** 那份 `docs/信贷-需求到详设-解决方案.md` 大方案**用户判定"落不了地"已弃用**——它假设了内网根本没有的东西(领域本体/制度库/向量RAG/claim账本框架/状态机闸口)。新方向只信内网唯一可信的两样:**代码仓库 + 数据库**。需求文档是"待核对的主张",代码+DB 是"真相"。本质:**拿需求文档逐条到代码+DB 找证据 → 改动点(挂 file:line / 表.字段)+ 需求问题(文档 vs 代码/DB 的缺口/冲突/歧义)**。不建领域大脑——领域知识就编码在代码和 DB 里。

底座几乎现成:需求文档解析复用 `src/attachments.js` 的 extractText(PDF/DOCX/XLSX);代码检索用 opencode serve 原生 Read/Grep/Glob;DB 用已有 `db-mcp`(db_columns_grep/db_schema/db_query);fan-out 用动态工作流卡(`src/window.js` spawnWorkflow,单主 Agent 连续上下文自拆+并行派子 Agent+自综合;旧 `src/orch.js` 已退役删除)。要建的是薄编排层 + 独立窗口。

**核心机制(2026-06-26 用户定死,技术深水):多Agent 对抗式分析需求文档。** 真实需求是脏数据(没格式/标准、诉求藏在吐槽里、多叫法、范围模糊、外部引用、跨页矛盾、V1残留),单个 opencode Agent 搞不定。机制:
- **让多个 Agent 各自独立、互不通气地读同一份文档**(角色分化:业务字面/数据派/流程派/挑刺对抗派/历史跨页派)。
- **分歧 = 歧义探测器**:全体读法一致→意思明确(合并解释,高置信);读法分裂→意思不明确,而**分裂的几种读法正好就是把歧义拆解清楚**(并列①②③给人选);仅 1 个(尤其挑刺派)发现→隐藏诉求/易漏;挑刺+历史派专门挖跨页矛盾/V1残留/未写假设。
- **对齐引擎=最难最值钱的核心**:按读法指向的原文 span/术语聚类、算一致度、置信度聚合,模糊时上"裁判 Agent"。span 重叠判定 + claim 聚类是真技术含量。
- **代码+DB 是消歧字典**:对分裂读法/拿不准的术语 fan-out grounder 查 db-mcp + grep 代码,收敛歧义 or 坐实缺口,每条挂 file:line/表.字段。
- **人工二次确认=唯一真相源**:机器只摊开三类清单(明确/不明确/矛盾),人认领/选读法/裁矛盾。
- **落档+越分析越上手**:只有人确认过的结论(术语归一/已决歧义/范围决策)才 append-only 落档,锚定文档版本+代码/DB ref;下次分析加载→settled 不再重复问。**这是"赚来的"记忆,不是被否掉的预建本体/领域大脑**——区别关键,别再提预建知识库。

**已锁定 UX(2026-06-26):** ①入口=独立「需求分析」窗口/向导 ②拆出后停下让人确认(人审闸口)③报告=交互面板,证据可点开(file:line→编辑器,表.字段→看 schema)④**彻底不碰邮件/outbox**(用户强调"跟邮件完全没关系")。

**已做的探索:** 用 visualize 交互画布给用户演示了 3 版原型(流程图→交互UI原型→"诚实版拆点"脏文档标注→多Agent对抗架构图);用户认可"诚实摊开脏数据 + 对抗分歧"方向。注:用户是强视觉型,要可点的真 UI 原型不要流程图。

**底座(几乎现成):** orch.js 的 fan-out + onBeforeBatch 人审闸 = spawn + 二次确认骨架;opencode serve 原生 Read/Grep = grounder 读代码;db-mcp = grounder 读 DB;attachments.js extractText 解析文档。**新要建的硬核:①多视角对抗阅读编排(角色分化+独立性)②对齐引擎(最难)③三类清单二次确认 UI ④落档知识库+回灌闭环。**

**How to apply:** 接着推时不要回头碰信贷大方案、不要引入邮件、不要预建领域本体。技术核心是对齐引擎。符合 [[project_personal_agent_focus]]。相关 [[project_intranet_mode]](DB/代码都在内网)、[[project_bocomhermes]]。
