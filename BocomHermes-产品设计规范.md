# BocomHermes — 产品设计规范（Design Spec v1.0）

> 目的：把BocomHermes 从零散的视觉尝试，收敛为一套可落地、可验收的产品设计。开发照此实现，设计照此评审。
> 范围：桌面端（Windows 优先）。产品 = 全能对话式智能体，以**可拖动的玻璃对话卡**为核心载体，底层走 OpenCode + 内网模型。
> 假设：内网研发场景优先；数据不出网；单人桌面；深色玻璃为主材质（理由见 §7.1）。

---

## 1. 设计原则（北极星）

| # | 原则 | 含义 | 设计含义 |
|---|---|---|---|
| 1 | 对话即界面 | 不用菜单和流程，你说人话 | 主界面只有"输入框 + 对话卡"，没有功能树 |
| 2 | 并行不打断 | 多任务并行，注意力不被夺走 | 卡片悬浮、独立、可随手摆放，绝不全屏霸屏 |
| 3 | 透明在场 | 浮在你的工作之上，是助手不是主角 | 玻璃材质 = 在场但不遮挡；克制尺寸 |
| 4 | 意图驱动 | 你给意图，它编排手段 | 不暴露"阶段/流程"，能力作为可选建议出现 |
| 5 | 可信可控 | 看得见它在做什么，敏感操作要你点头 | 实时活动、出处可溯、写/执行前确认 |
| 6 | 克制的科技感 | 科技感来自光、深度、动效，不是堆装饰 | 一处辉光、一条高光、一个渐变签名色，足矣 |

## 2. 用户与场景

**主用户**：内网软件研发工程师（如信贷系统 Java/MyBatis 开发）。

**核心 JTBD**：
> 当我在改代码 / 查需求时，我想随手叫一个**懂我这套代码库**的助手，并行帮我评审、定位影响面、补测、查库，**不打断我手头的事**，且**代码不出内网**。

**一句话场景**：
> 改授信审批时,顺手 `Ctrl+Shift+Space`,说"看下这个改动有没有坑",一张卡浮出来流式给结论;我继续写代码,需要时点卡里的"补单测";同时另开一张卡查影响面。四张卡并排在副屏,我只在关键处点头。

**非目标（先不做）**：泛办公全场景、移动端、多人协作。先把研发对话做到极致。

## 3. 核心交互模型

```
        ┌── 全局启动器（Launcher）── 热键唤起的输入框，开"新对话"
        │
你 ─────┤── 对话卡（Card）── 一张卡 = 一个持续会话；自带输入框，可继续聊
        │         · 可自由拖动 / 并行 / 收纳
        │         · 卡内：消息流 + 结果块 + 建议气泡 + 权限确认
        │
        └── 卡坞（Hub，后期）── 所有卡/历史的总览与召回
```

三条铁律：
- **启动器开新对话，卡内继续对话**（解决"每次都开新窗"）。
- **能力是建议不是流程**：卡片答完给可选"下一步"气泡，点不点都行。
- **写/执行需点头**：只读自动放行，改文件/跑命令在卡内弹确认。

---

## 4. 信息架构与界面清单

扁平，无深层导航。共 5 个界面（surface）：

| 界面 | 角色 | 形态 |
|---|---|---|
| 启动器 Launcher | 开新对话的入口 | 屏幕上方悬浮玻璃输入条，热键唤起 |
| 对话卡 Card | 一个会话的全部 | 无边框玻璃窗口：标题栏 + 消息流 + 输入栏 |
| 权限确认 Permission | 敏感操作把关 | 卡内内联条（非弹窗），允许/拒绝 |
| 卡坞 Hub（后期） | 总览/召回所有卡 | 可收起的卡片网格 + 历史检索 |
| 设置 Settings | 模型/热键/材质 | 独立小面板 |

---

## 5. 关键流程

1. **起对话**：热键 → 启动器 → 输入回车 → 生成卡 → 流式回复。
2. **并行**：再热键 → 启动器 → 第二张卡；自由拖放并排。
3. **继续**：在卡底输入框追问 → 同会话续写。
4. **批准操作**：卡内出现"请求执行 `bash: git diff`" → 允许一次 / 总是 / 拒绝。
5. **消费结果**：结果块（diff / 清单 / 文件引用）→ 复制 / 应用 / 打开文件 / 派生新卡。
6. **收纳召回**：卡片最小化进卡坞 → 需要时召回（后期）。

## 6. 界面状态矩阵（每个界面都要设计这些态）

| 界面 | 空/初始 | 进行中 | 完成 | 错误 | 特殊 |
|---|---|---|---|---|---|
| 启动器 | 占位提示语 | — | — | 热键冲突提示 | 聚焦辉光 |
| 对话卡 | 首条消息 + 思考中 | 流式打字 + 状态转圈 | 状态对勾 + 结果块 | 模型不可达/超时重试 | 长内容滚动、权限内联、空会话 |
| 权限条 | — | 等待你决定 | 已允许/已拒绝（淡出） | — | "总是允许"记忆 |
| 卡坞 | 无卡占位 | — | 卡片网格 | — | 历史检索 |

> 规矩：**任何会"转圈"的地方都要有完成态和错误态**；任何流式输出都要可中断（停止按钮）。

## 7. 设计系统（Design Tokens）

### 7.1 玻璃材质规范 · "墨玻璃 Ink Glass"
- **定调**：深色烟熏玻璃 + 冷光描边。**为什么深色**：科技感/HUD 观感来自"暗底 + 发光"，浅白玻璃做不出；且深色作底能保证浅色字在任意桌面上可读（见 §10）。
- **双主题（已定，并行发布）**：**墨玻璃（深）+ 浅磨砂（浅）**，用户随时切换、记住选择，默认浅色；**一套设计令牌、两套皮肤**，由 `html[data-theme="dark|light"]` 驱动（实现见 `BocomHermes-shell/ui/glass.css`）。
- **结构（三层）**：
  1. 底纹理：`linear-gradient(158°, rgba(26,30,52,.66), rgba(13,16,30,.56))` —— 半透深色，桌面隐约透出。
  2. 边缘：1px `rgba(255,255,255,.14)` 描边 + 外发光 `0 0 26px rgba(80,130,255,.22)`。
  3. 高光：顶部一条 1px 渐变扫线（HUD 质感）+ `inset 0 1px 0 rgba(255,255,255,.16)`。
- **可读性保证**：底纹理不透明度不得低于 ~50%，确保字够清（这是硬约束，不是审美选项）。

### 7.2 色彩
| 类别 | Token | 值 | 用途 |
|---|---|---|---|
| 底 | `--glass-1/2` | `rgba(26,30,52,.66)` / `rgba(13,16,30,.56)` | 玻璃底 |
| 字-主 | `--txt` | `rgba(237,241,255,.95)` | 标题/正文 |
| 字-次 | `--txt2` | `rgba(237,241,255,.60)` | 辅助 |
| 字-弱 | `--txt3` | `rgba(237,241,255,.38)` | 占位/遥测 |
| 主强调 | `--cyan` | `#46e0ff` | 进行中/链接/聚焦 |
| 辅强调 | `--violet` | `#9b8cff` | 品牌点缀 |
| **签名渐变** | `--brand` | `linear-gradient(135°,#9b8cff,#46e0ff)` | 星标、发送键、品牌时刻 |
| 成功 | `--green` | `#54e0a8` | 完成 |
| 警告 | `--amber` | `#ffcf6b` | 需注意 |
| 危险 | `--red` | `#ff7a8a` | 必改/拒绝 |

> 强调色**只用于状态与品牌时刻**，不滥用；一屏最多见到 1–2 个强调色。

### 7.3 字体与排版尺度
| 级别 | 字号/字重 | 用途 |
|---|---|---|
| 标题 | 13 / 600 | 卡标题 |
| 正文 | 13 / 400 | 消息、输入 |
| 次要 | 12 / 400 | 说明 |
| 遥测 | 11 / 400 mono | 会话号、时间、工具名 |
- 字体：`Segoe UI Variable`/系统无衬线；遥测用等宽。**两档字重**（400/600），不用更重。

### 7.4 间距 / 圆角 / 层级
- 间距基数 **4px**：常用 8 / 12 / 16 / 24。
- 圆角：卡片 16、气泡 13、药丸/按钮 999、小标签 6–7。
- 透明留白：窗口内留 **24px** 透明边，给发光与投影完整渲染空间。
- 层级（elevation）：卡片 = 发光 + 投影双层；聚焦态加强发光；拖动态投影更深（手感"被拎起")。

### 7.5 动效
| 名称 | 时长 / 缓动 | 用途 |
|---|---|---|
| 卡弹入 pop | 220ms / cubic(.2,.85,.25,1) | 新卡/聚焦出现 |
| 流式打字 | ~16ms/字 | AI 回复逐字 |
| 状态切换 | 200ms | 转圈→对勾 |
| 边缘呼吸 glow | 2.4s 循环 | 进行中卡的活着感 |
| 悬停反馈 | 120ms | 按钮/可点元素 |
- 原则：**快、有目的、不挡路**；尊重系统"减少动态效果"（reduced-motion 时关闭呼吸/弹入）。

### 7.6 图标
- 线性图标（stroke 2.4），进行中用 `--cyan`、完成用 `--green`；品牌星标用签名渐变。

## 8. 组件规范

| 组件 | 关键点 |
|---|---|
| 启动器条 | 左侧渐变星标（兼拖动手柄）+ 输入 + 热键提示；聚焦辉光；`Esc` 收起；`Enter` 开卡 |
| 对话卡 | 标题栏（状态/标题/会话号/关闭）+ 消息流（可滚）+ 输入栏；**标题栏拖动**，消息区可滚/可选 |
| 消息气泡 | 你=签名色淡染靠右，AI=浅白靠左；圆角带"尾角"；支持流式 |
| 流式指示 | 转圈 + "停止"按钮（可中断）|
| 结果块 | diff / 清单 / 文件引用，带"复制 / 应用 / 打开 / 另开卡"操作 |
| 建议气泡 | 答后给 2–3 个可选下一步药丸，点或不点皆可 |
| 权限条 | 内联（非弹窗）："请求 `bash:git diff`"→ 允许一次 / 总是 / 拒绝 |
| 状态徽标 | 运行(青转圈)/完成(绿勾)/排队(灰钟)/出错(红) |
| 卡坞（后期）| 所有卡缩略网格 + 历史检索 |

## 9. 微交互
- 新卡**弹入**并自动聚焦输入；首条消息即你在启动器输入的话。
- AI 回复**逐字流式**，消息区自动吸底；可随时"停止"。
- 进行中卡**边缘呼吸辉光**；完成时辉光收敛、状态转绿勾。
- 拖动时投影加深（被拎起感）；松手回落。
- 聚焦输入：边缘辉光增强（青/紫）。

---

## 10. 可读性与无障碍（★容易被忽略，但很关键）
- **玻璃可读性**：卡片透明,但背后桌面不可控。必须有"底纹理 scrim"保证对比度——底不透明度 ≥ ~50%,文字永远落在 scrim 上,而非直接压在桌面像素上。透明是为了"在场感",不是牺牲可读。
- **键盘优先**：全局热键唤起；`Enter` 发送、`Esc` 收起/最小化；卡间可用快捷键循环（后期）。
- **对比度**：浅色字与墨玻璃底对比 ≥ WCAG AA。
- **减少动态**：reduced-motion 下关弹入/呼吸/逐字（直接出全文）。
- **缩放/多屏/高 DPI**：尺寸用相对值；新卡在当前光标所在屏生成；防止卡片落到屏外。

---

## 11. 边界情况
- 超长输出 → 卡内滚动 + "展开/折叠"；不无限拉高窗口。
- 卡片过多（20+）→ 自动错落 + 卡坞收纳。
- 模型慢/不可达 → 明确"连接中/超时"态 + 重试，不空转。
- 热键被占用 → 启动时检测并提示改键。
- 拖到屏外 → 自动回收到可视区。
