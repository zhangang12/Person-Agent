# BocomHermes — 个人智能体产品蓝图（Vision Blueprint）

> 一句话愿景：**让每个内网工程师，都拥有一支并行工作、记得住一切、数据永不出门的 AI 工作团队。**
> 北极星：把"我自己一个人干"变成"我带着一支 AI 团队干"——同样的人，吞吐与质量提升一个量级。
> 底座原则不变：BocomHermes 是 **opencode serve 的外层增强壳**，只在外面长能力，不碰 opencode 自身配置。

---

## 1. 产品演进：三级跳

| 层级 | 形态 | 关键词 | 当前进度 |
|---|---|---|---|
| **L1 一个好用的 AI 同事** | 透明玻璃对话卡，单卡单会话 | 顺手、可信、能干活 | ✅ 已基本成型（流式/权限/Markdown/多项目隔离/打包） |
| **L2 一支并行的 AI 团队** | 多卡协作、角色分工、目标自动编排 | 并行、编排、分身 | 🔜 下一阶段主战场 |
| **L3 一个常驻的研发 OS** | 主动感知、记忆沉淀、全工具链打通、技能平台 | 常驻、自治、生态 | 🌅 愿景 |

核心叙事：opencode 给了"一个能用工具的大脑";BocomHermes 把它**复制成一支团队、装上记忆、接进你的工具、并常驻在你的桌面**。

---

## 2. 能力底座：opencode 给了什么，我们在上面加什么

| opencode 已提供（直接用） | BocomHermes 在外层加的"增强" |
|---|---|
| 会话 / 工具调用（读写/grep/bash） | 玻璃卡 UI、多卡并行、画布编排 |
| 多会话并发 | 按项目隔离的 serve 连接池（✅）、智能体团队调度 |
| 事件流 / 权限 | 实时流式渲染、内联权限、**审计与回放** |
| MCP（接外部工具） | 内网系统集成市场（GitLab/Jira/DB/CI） |
| 自定义 Agent / Command | 角色化数字同事、技能市场 |
| 多模型 | 模型路由（小任务小模型、长上下文大模型） |

> 一句话：**opencode 负责"会不会做",BocomHermes 负责"怎么用得爽、管得住、连得通、记得牢"。**

## 3. 五大能力支柱（+ 治理 + 平台）

### 支柱一 · 交互（让"用"变成第二天性）
- **选中即问 / 截图即问**：在任意 IDE/网页选中代码或框选屏幕 → 热键直接开卡带上下文。`[next]`
- **拖入即上下文**：把文件/文件夹/报错日志拖进卡片当输入。`[next]`
- **语音输入**：说一句话开一张卡。`[later]`
- **卡片画布（招牌）**：无限缩放的空间画布，卡片可分组、连线成管道。`[later]`
- **富结果**：diff 一键应用、`文件:行` 跳转（🔜）、Mermaid/表格/图表渲染、代码可运行。`[next]`

### 支柱二 · 编排（从一个分身到一支团队）
- **角色化数字同事**：需求分析师 / 架构师 / 开发 / 测试官 / 评审官，各有专属提示与权限。`[next]`
- **一键 fan-out**：「让三个视角评审这段」→ 并行开 3 张卡 → 一张"汇总卡"合并结论。`[next]`
- **目标→自动编排（招牌）**：给一句目标，主管智能体拆解 → 并行/串行派生子卡 → 在关键节点找你拍板。`[later]`
- **技能/配方（Recipes）**：把常用多步流程（定位→改→补测→评审）存成可复用技能，一键起一套卡。`[next]`
- **辩论 / 投票**：多智能体对抗式互评，多数通过才算数（治幻觉）。`[later]`

### 支柱三 · 主动（从"我问它"到"它提醒我"）
- **触发器**：文件变更 / git 提交前 / 定时 / 剪贴板 → 自动起任务。`[later]`
- **常驻监视卡**：盯日志、盯告警、盯某接口，有事才亮。`[later]`
- **晨间简报（招牌）**：每天早上汇总昨夜变更/告警/你的 TODO，给一张简报卡。`[later]`
- **后台任务 + 托盘徽标**：长任务后台跑完通知你。`[next]`

### 支柱四 · 记忆与知识（越用越懂你）
- **个人记忆库**：记住你的偏好、过往决策、踩过的坑，自动注入上下文。`[next]`
- **项目知识底座**：自动维护并复用术语词典 / 能力地图（你已有的 Wiki 体系），RAG 检索内网文档。`[next]`
- **卡坞 / 历史**：所有会话可检索、可恢复、可继续。`[next]`
- **学你的风格**：命名、代码风格、评审口味——自动套用。`[later]`

### 支柱五 · 行动与集成（接进你真实的工具链，靠 MCP）
- **内网系统**：GitLab/SVN、Jira/禅道、数据库（只读）、Confluence、CI/CD、制品库。`[next/later]`
- **一键动作**：生成 commit / 开 MR / 建工单 / 跑流水线 / 查表。`[later]`
- **编辑器 & 终端联动**：跳转 `文件:行`、把 diff 应用到工作区、命令带确认执行。`[next]`

### 横切一 · 治理与可信（银行内网的生命线）
- **权限策略中心**：按工具/项目配"读自动·写确认·危险拒"，可设白名单。`[next]`
- **审计与回放**：智能体干了什么（谁/何时/动了哪些文件/跑了什么）全程留痕、可回放。`[next]`
- **数据边界**：强制不出网、敏感信息脱敏后再喂模型、命令沙箱执行。`[next]`
- **用量与预算**：每张卡/每个项目的 token 用量与上限。`[later]`

### 横切二 · 平台化（从工具到生态）
- **技能市场**：团队内共享技能/配方/角色/提示词（信贷团队的评审技能、SQL 分析技能…）。`[later]`
- **团队版**：共享记忆与技能、统一策略与审计，但每人数据各自留本地。`[vision]`
- **本地 API**：让其它内网工具/IDE 插件触发BocomHermes 卡片。`[later]`

## 4. 五个招牌功能（产品的"记忆点"）

1. **卡片画布 Agent Canvas**：把"多卡并行"升级成空间化的智能体工作台——卡片连线成流水线，一眼看到整支团队在并行干活。这是BocomHermes 的视觉灵魂。
2. **目标→团队 Goal-to-Team**：一句话目标 → 自动派生一支协作的子卡团队，你只在闸口拍板。把"全生命周期"从流程变成"召之即来的团队"。
3. **数字同事 Personas**：有名字、有专长、记得过往的角色智能体，像真同事一样被你"指派任务"。
4. **晨间简报 Morning Briefing**：常驻、主动——每天给你一张"昨夜发生了什么 + 今天该干什么"的卡。
5. **内网唯一可信 AI**：数据不出门 + 私有化模型 + 全程审计——这不是功能，是**护城河**：合规环境里别的 AI 进不来，BocomHermes 是唯一选项。

## 5. 演进路线（Horizon）

| 阶段 | 目标 | 代表功能 |
|---|---|---|
| **H1 打磨单兵（近）** | 单卡体验做到极致 | 选中即问、富结果(diff/跳转)、卡坞历史、设置面板、后台任务 |
| **H2 组成团队（中）** | 从一个分身到一支团队 | 角色同事、一键 fan-out、技能配方、目标→团队、卡片画布 |
| **H3 常驻自治（远）** | 主动 + 记忆 + 集成 | 触发器、晨间简报、记忆库、MCP 接内网系统、审计回放 |
| **愿景 平台化** | 从工具到生态 | 技能市场、团队版、组织级研发数字员工平台 |

## 6. 护城河与定位

- **合规即壁垒**：数据不出网 + 私有化模型，正面满足金融/政企内网硬约束——公网 AI 结构性进不来。
- **连续记忆**：单一智能体贯穿全程 + 个人/项目记忆，越用越准——别人是工具，BocomHermes 是"懂你的同事"。
- **复用既有资产**：把你已沉淀的 Wiki/术语/能力地图变成智能体的长期记忆。
- **站在 opencode 上**：不重造引擎，借开源演进红利，自身聚焦"体验/编排/治理/集成"。

---

## 7. 北极星指标 & 下一步建议

**北极星指标**：单位工程师**每周被 AI 完成/加速的真实研发任务数**（不是对话条数，是"真改了/真定位了/真评审了"的任务）。

**作为 PM，我建议的下一步优先级**（H1 内、性价比最高的三件）：
1. **卡坞 / 历史 + 后台任务**：让会话不丢、长任务能后台——这是"敢重度使用"的前提。
2. **富结果：diff 应用 + `文件:行` 跳转**：让输出从"能看"变"能用"，直接闭环到代码。
3. **角色同事 + 一键 fan-out**：迈出 L1→L2 的第一步，让"并行的玻璃卡"真正变成"并行的团队"。

> 一句话收尾：现在你有了"一个能干活的 AI 同事";蓝图就是把它变成**一支懂你、并行、可信、常驻的 AI 团队**——而且因为数据不出门,它会是你内网里**唯一**的那一个。

## 9. 关键指标（KPI）

- **北极星**：单位工程师每周"被 AI 真实完成 / 加速的研发任务数"。
- **H1（单兵）**：周活跃卡片数；单卡产出被采纳率；从提问到可用结果的平均时长。
- **H2（团队）**：并行卡片占比；一次 fan-out / 团队任务的采纳率；返工率下降。
- **H3（常驻）**：主动任务（触发 / 简报）触达与采纳率；记忆命中率（注入上下文被真正用到）。
- **横切（红线，必须满足）**：写/执行操作经确认率 = 100%；操作可追溯率 = 100%；数据出网事件 = 0。

## 10. 风险登记（Risk Register）

| 风险 | 影响 | 对策 |
|---|---|---|
| 模型质量参差 | 产出不可信 | 结论带出处 + 人在回路 + 对抗式互评（多数通过） |
| 误删 / 误改 / 跑坏命令 | 破坏代码 | 只读默认放行 + 写/执行确认 + 危险命令黑名单 + 审计回放 |
| 数据出网 | 合规事故 | 私有化模型 + 全程不出网 + 脱敏（opencode 层把关，外壳不碰配置） |
| 范围蔓延 | 做不完、做不深 | 严格按 Horizon 推进，先把 H1 闭环再上 H2 |
| opencode 升级破坏集成 | 功能失效 | 端点以本机 `/doc` 为准 + 版本固定 + 升级先在测试机验证 |
| 过度承诺"全能" | 预期落空 | 先把"研发对话/团队"做到极致，再谈泛办公 |

## 11. 对标主流个人智能体与设计借鉴（Hermes / OpenClaw）

参考 2025–2026 主流自治个人智能体——Nous **Hermes**（一核多端 · 跨会话自进化 · 解决难题后自动写"技能文档"）与 **OpenClaw**（活在 IM 里 · 技能市场 ClawHub · heartbeat 心跳）——取其共性模式，按**内网银行**约束适配。底座不变：BocomHermes 仍是 opencode serve 的外层封装，借鉴的是形态与模式。

**它们的共性 6 模式**：① 一核多端 ② 模型无关 + 自托管 ③ 技能即扩展 + 技能市场 ④ 心跳 / 主动 ⑤ 持久记忆 + 自进化 ⑥ 真动作（跨工具执行）。

**借鉴后补进BocomHermes 的 4 个设计**（已进 §8 Backlog，标 ★借鉴）：
1. **自进化技能（最该抄）**：一次有价值的任务（评审/定位/修复）完成后，自动把"怎么做的"蒸馏成可复用技能文档，存进项目知识库——与BocomHermes 记忆体系、已有信贷 Wiki 天然合一，越用越准。
2. **心跳 / 主动调度**：常驻心跳按计划唤醒，监控 git/CI/日志/工单，有事才主动给你一张卡/一条 IM（默认只读，动手仍需点头）。
3. **内网 IM 网关**：企业微信 / 钉钉发消息即可指挥，**仅内网、绝不连公网 IM**；网关与桌面壳共用同一 serve 池 / 权限 / 技能。
4. **内网技能市场**：团队内共享技能 / 配方 / 角色，只在内网流转。

**对标定位表**

| 维度 | Hermes | OpenClaw | **BocomHermes** |
|---|---|---|---|
| 形态 | CLI+TUI+Electron+IM 网关 | IM 网关为主（多平台） | **桌面玻璃卡（并行）** + 启动器（+ 可扩 CLI/REST/内网 IM） |
| 模型/数据 | 200+ 模型，多为云 | 自选模型，本机 | **内网私有化，数据不出门** |
| 扩展 | 自写技能文档（自进化） | AgentSkill + ClawHub 市场 | opencode agent/cmd/MCP + 内网技能市场（规划） |
| 主动 | 自治运行 | heartbeat 心跳 | 借鉴升为一等（心跳/简报） |
| 记忆 | 跨会话自进化 | 本地 | 个人/项目记忆 + 自进化技能 |
| 定位 | 自托管自进化通用 agent | IM 里的万能助手 | **内网原生 · 研发团队 · 合规可信** |

**差异化（抄共性，守独有）**：BocomHermes 守住三个别人给不了的点——① **并行玻璃卡 UI**（不是聊天框/IM，是桌面上可拖动、可并行、可编排的"玻璃卡团队"）；② **内网原生 · 数据不出门**（唯一能进银行内网的那类）；③ **复用既有资产**（把信贷 Wiki/术语/能力地图变成自进化技能的起点）。

> 一句话：BocomHermes = **内网版的 Hermes / OpenClaw**，但把"IM 形态"换成"**并行玻璃卡团队**"，把"云"换成"**数据不出门**"，并站在 opencode 上。
> 来源：[NousResearch/hermes-agent](https://github.com/nousresearch/hermes-agent)、[OpenClaw docs](https://docs.openclaw.ai/)。
