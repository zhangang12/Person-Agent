# 天枢 vs 主流个人智能体（hermes / openclaw）：对标与设计借鉴

> 目的：参考 2025–2026 主流"个人自治智能体"（Nous **Hermes**、**OpenClaw**）的成熟设计模式，取其精华、按**内网银行**约束适配，补进天枢蓝图。
> 底座原则不变：天枢是 **opencode serve 的外层封装**，借鉴的是"形态与模式"，不是换引擎。

---

## 1. 它们是什么（简述）

- **OpenClaw**：自托管、模型自选的自治智能体；**活在你已有的 IM 里**（Telegram/Slack/Teams/Signal… 多平台网关），靠 **AgentSkill 技能 + 技能市场（ClawHub）** 扩展；有 **heartbeat 心跳**——定时自动唤醒、监控文件系统/数据库，无需人催。
- **Hermes（Nous Research）**：**一套 agent 核心，多端同跑**（CLI / TUI / Electron 桌面 / 20+ IM 网关）；模型无关；**跨会话持久记忆、自我进化**——解决难题后**自动写一份可复用"技能文档"**，越用越强；支持多种执行后端（本地/Docker/SSH…）。

> 来源：Hermes（nousresearch/hermes-agent）、OpenClaw（docs.openclaw.ai）。文中只采纳其设计模式。

---

## 2. 它们的共性设计模式（值得借鉴的 6 条）

1. **一核多端**：同一 agent 核心，跑在 CLI / TUI / 桌面 / IM 网关多个入口。
2. **模型无关 + 自托管**：模型自选、跑在自己基础设施上（隐私/可控）。
3. **技能即扩展单元 + 技能市场**：能力以"技能"为粒度封装、可分享、可复用。
4. **心跳 / 主动**：定时自动唤醒、监控、无需人催地干活。
5. **持久记忆 + 自我进化**：跨会话记忆；把解决过的难题**自动沉淀成可复用技能**。
6. **真动作**：跨工具执行（命令/文件/网络/系统），不止聊天。

## 3. 天枢对标：已有 / 该补 / 怎么为内网适配

| 模式 | 天枢现状 | 借鉴动作（按内网适配） |
|---|---|---|
| 一核多端 | 桌面玻璃卡 + 启动器；CLI/REST 脚手架（desktop-agent） | **加内网 IM 网关**（企业微信/钉钉/内网 IM）；TUI/CLI 复用同一 serve 池 |
| 模型无关+自托管 | ✅ 内网私有化模型、数据不出门 | 已是强项——继续作为护城河强调 |
| 技能 + 市场 | opencode agent/command/MCP；蓝图有"技能市场" | **技能/配方做成一等公民** + **内网技能市场**（团队共享，不连公网） |
| 心跳/主动 | 蓝图有"触发/晨间简报"（未做） | **把"心跳"升为一等概念**：定时唤醒、监控 git/CI/日志/工单、主动简报告警 |
| 持久记忆+自进化 | 蓝图有"记忆支柱"（未做） | **加"自进化技能"**：任务完成后自动蒸馏成技能文档，存进项目知识库 |
| 真动作 | ✅ 经 opencode 工具调用 + 权限内联 | 继续；接内网系统（GitLab/Jira/只读 DB）走 MCP |

## 4. 借鉴后新增到设计的 4 个重点

### 4.1 内网 IM 网关（一核多端 · 适配内网）
- 在 **企业微信 / 钉钉 / 内网 IM** 上挂一个机器人：你发消息 → 网关转给本机/内网的 serve 池 → 跑完回贴结果。
- 让你"离开工位也能指挥"——但**只接内网 IM**，绝不连 WhatsApp/Telegram 这类公网。
- 技术上：网关是又一个"serve 的外层客户端"，和桌面壳共用同一套会话/权限/技能。

### 4.2 心跳 / 主动调度（heartbeat，升为一等概念）
- 一个常驻"心跳"：按计划唤醒智能体去**监控**（昨夜 git 提交、CI 结果、关键日志、新工单），有事才**主动**给你一张卡/一条 IM。
- 典型：**晨间简报**、**提交前自检**、**告警根因预判**。
- 安全：主动任务默认只读 + 只通知,要动手仍需你点头。

### 4.3 自进化技能（self-improving skills，最值得抄）
- 一次有价值的任务（评审/定位/修复）完成后,智能体**把"怎么做的"蒸馏成一份可复用技能文档**(输入→步骤→产出),存进**项目知识库**(正好复用你已有的 `docs/` 与术语/能力地图)。
- 下次同类任务直接调这份技能 → **越用越准、越用越快**。
- 与天枢记忆体系天然合一:技能文档 = 团队级长期记忆。

### 4.4 内网技能市场（skill hub · 团队版）
- 团队内共享技能/配方/角色(信贷团队的"评审规约技能""SQL 口径技能"…),**只在内网流转**。
- 个人版用本地技能库;团队版用内网共享库 + 统一权限/审计。

## 5. 对比定位表

| 维度 | Hermes | OpenClaw | **天枢 Tianshu** |
|---|---|---|---|
| 形态 | CLI+TUI+Electron+IM 网关 | IM 网关为主(多平台) | **桌面玻璃卡(并行)** + 启动器(+ 可扩 CLI/REST/内网IM) |
| 模型/数据 | 200+模型,多为云 | 自选模型,本机 | **内网私有化,数据不出门** |
| 扩展 | 自写技能文档(自进化) | AgentSkill + ClawHub 市场 | opencode agent/cmd/MCP + 内网技能市场(规划) |
| 主动 | 自治运行 | heartbeat 心跳 | 借鉴升为一等(心跳/简报) |
| 记忆 | 跨会话自进化 | 本地 | 个人/项目记忆 + 自进化技能(借鉴强化) |
| 定位 | 自托管自进化通用 agent | IM 里的万能助手 | **内网原生 · 研发团队 · 合规可信** |

## 6. 结论：天枢的差异化（抄共性，守独有）

借鉴它们的 **一核多端 / 心跳主动 / 自进化技能 / 技能市场**;但天枢守住三个**别人给不了**的独有点:

1. **并行玻璃卡 UI** —— 不是聊天框、不是 IM,而是桌面上一支可拖动、可并行、可编排的"玻璃卡团队"。这是天枢的形态身份。
2. **内网原生 · 数据不出门** —— hermes/openclaw 多偏云/公网 IM;天枢是**唯一能进银行内网**的那一类。
3. **复用既有资产** —— 把你已沉淀的信贷 Wiki/术语/能力地图变成"自进化技能"的起点。

> 一句话:天枢 = **内网版的 hermes/openclaw**,但把"聊天/IM 形态"换成"**并行玻璃卡团队**",把"云"换成"**数据不出门**",并站在 opencode 上。
